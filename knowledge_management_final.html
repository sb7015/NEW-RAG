Multi-Division Knowledge Management System

Intelligent Document Retrieval with AWS Bedrock & Agentic RAG Architecture

1Project Overview

Problem Statement:

The organization has multiple divisions, each maintaining extensive Standard Operating Procedures (SOPs) and process documents stored in SharePoint folders. Employees currently spend 10+ minutes manually searching through documents to find specific information, leading to inefficiency and potential errors.

Proposed Solution:

Build an AI-powered Knowledge Management System that allows employees to ask questions in natural language and receive accurate answers with citations to source documents. The system uses a two-agent architecture powered by AWS Bedrock to ensure high-quality, verified responses.

Key Features:

  • Natural language question-answering interface
  • Multi-division document support with access control
  • Automatic citation with source document links
  • Support for large PDFs and scanned documents
  • Two-agent verification to minimize hallucinations

üìÅ Step-by-Step Connection Process

  • Create Azure AD App Registration
    In Azure Portal, register a new application with SharePoint API permissions (Sites.Read.All, Files.Read.All)
  • Generate Client Secret
    Create a client secret in the app registration and note down Client ID, Secret, and Tenant ID
  • Configure Amazon AppFlow
    In AWS Console ‚Üí AppFlow ‚Üí Create Flow ‚Üí Select SharePoint as Source
  • Authenticate with SharePoint
    Enter SharePoint Site URL, Client ID, Client Secret, and Tenant ID in AppFlow connection settings
  • Map Source to Destination
    Source: SharePoint folder path (e.g., /sites/CompanyDocs/SOP/)
    Destination: S3 bucket (e.g., s3://knowledge-base-docs/division-name/)
  • Set Sync Schedule
    Configure hourly or daily sync schedule with incremental updates enabled
  • Enable Metadata Mapping
    Map SharePoint metadata fields (Modified Date, Author, Division) to S3 object tags
  • Test Connection
    Run a test flow to sync 5-10 sample documents and verify they appear in S3
  • Configure Bedrock Knowledge Base
    Point Knowledge Base data source to the S3 bucket for automatic indexing

4End-to-End System Flow
Step 1 Data Ingestion
Amazon AppFlow automatically syncs documents from SharePoint to S3 buckets. Documents are organized by division with metadata (name, date, division).
Step 2 Document Processing
AWS Textract performs OCR on scanned PDFs. Bedrock Knowledge Base automatically chunks documents (semantic chunking), generates embeddings (Titan v2), and indexes in OpenSearch Serverless.
Step 3 User Query
User enters question in Streamlit chat interface with optional division filter. Query is sent to API Gateway.
Step 4 Agent A: Document Retrieval (Claude Sonnet 4.5)
Queries Bedrock Knowledge Base, retrieves top 5-10 relevant chunks with metadata (document name, page number, relevance score). Generates S3 pre-signed URLs for citations.
Step 5 Agent B: Synthesis & Verification (Claude Opus 4.5)
Receives chunks from Agent A, synthesizes coherent answer, verifies citations match content (hallucination check), generates confidence score.
Step 6 Response Display
Streamlit displays answer in chat format with inline citations. Each citation is clickable (S3 pre-signed URL with 1-hour expiry). Shows confidence score and processing time.
